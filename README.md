Задача 2. Выбор платформы виртуализации

Задача: высоконагруженная база данных MySql, критичная к отказу  
Тип платформы: физические серверы  
Критерии: нужна максимальная производительность дисковой подсистемы и памяти. Важно, чтобы не было дополнительных затрат ресурсов на гипервизор и исключать влияние соседних облачных сервисов.

Задача: Разные web-приложения  
Тип платформы: виртуализация на уровне ОС (контейнеры)  
Критерии: важно разместить много приложений на одном сервере, быстро запускать и масштабировать сервисы. Также важно удобно изолировать зависимости, например, поддерживать разные версии библиотек для разных приложений.

Задача: Windows-системы для использования бухгалтерским отделом  
Тип платформы: паравиртуализация (гипервизор)  
Критерии: нужна возможность быстро создавать снимки системы перед обновлениями. Удобно организовывать удалённые рабочие места (VDI или RDP). А также гибко управлять ресурсами ОС без необходимости менять железо.

Задача: системы, выполняющие высокопроизводительные расчёты на GPU
Тип платформы: физические серверы  
Критерии: требуется прямой доступ к видеокартам, максимальная скорость обмена данными между CPU и GPU. Важно избежать проблем с передачей специфичных драйверов в виртуальную среду.

Итоги:  
Если важна высокая производительность с оборудованием (БД, GPU) — лучше выбрать физические сервера.  
Если нужна гибкость, быстрая восстановимость и изоляция целых ОС (например, бухгалтерия) — подойдет паравиртуализация.  
Если нужно запускать много лёгких сервисов (веб-приложения) — лучше использовать контейнеризацию.
